ROS 2와 전용 로봇 툴박스 없이 MATLAB, Simulink, Stateflow로 실내 배송 AMR의 핵심을 직접 구현했다. 목표는 멋진 경로 그림 한 장이 아니었다. 센서가 늦거나 빠지고, 장애물이 갑자기 나타나고, 배터리가 부족해도 상위 상태기계와 하위 수치 알고리즘이 각자의 책임을 지키며 끝까지 움직이는 수직 절편을 만드는 것이었다.

이 연재는 완성된 결과부터 역으로 설명하지 않는다. 실제로 만들며 부딪힌 의존 순서대로 수학에서 센서, 지도, 계획, 감독제어와 검증까지 올라간다.

```mermaid
flowchart LR
    A["좌표·시간"] --> B["차동구동"]
    B --> C["LiDAR"]
    C --> D["지도·위치추정"]
    D --> E["전역·지역계획"]
    E --> F["Stateflow 감독제어"]
    F --> G["안전·복구"]
    G --> H["통합·회귀검증"]
```

## 이 프로젝트의 현재 수직 절편

```text
Hierarchical/Parallel Stateflow Supervisor
                    ↓
Occupancy Grid → A* Global Planner
                    ↓
2D LiDAR → Local Costmap → DWA Local Planner
                    ↓
Independent Safety Gate
                    ↓
Differential-Drive Plant → Playback UI and Logs
```

사무실, 병원 복도, 물류 창고의 합성 지도에서 정상 배송·돌발 장애물·배터리 부족·잘못된 길을 조합했다. Scenario Lab 12개와 통합 Stateflow 모델 12개 조합이 모두 통과했고 최종 위치 오차는 약 `0.080 m` 이하였다. 이 수치는 교육용 시뮬레이션 결과이며 실제 로봇의 안전 성능이나 인증을 뜻하지 않는다.
