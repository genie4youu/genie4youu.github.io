## 1. 등가성을 따지는 이유

01편에서 ADRC의 부모는 PID라고 했다. 그렇다면 정말 다른 물건인가, PID를 다르게 포장한 것인가. 이걸 명확히 하면 두 가지를 얻는다. 하나는 현장 설득이다. "ADRC는 필터를 잘 단 PID"라고 말할 수 있으면 PID만 아는 사람에게 신뢰를 준다. 다른 하나는 디버깅 직관이다. ADRC가 이상하게 굴 때 등가 PID로 환산하면 어디가 문제인지 짚인다.

## 2. 결론 — 필터 달린 2자유도 PID

> "Linear ADRC is equivalent to a PID controller with a **set-point weighting** and a **measurement filter**."
> — [arXiv 2501.11374](https://arxiv.org/abs/2501.11374)

주파수 영역에서 LADRC의 전달함수를 정리하면 PID에 설정값 가중과 측정 필터가 붙은 형태로 떨어진다.

| ADRC 요소 | PID 세계의 대응 |
| --- | --- |
| ESO의 외란 추정 | 적분 작용(I), 그 이상 |
| ESO의 속도 추정 | 미분 작용(D), 측정 필터 내장 |
| ESO 전체 | 측정 필터 |
| TD·설정값 처리 | 설정값 가중, 2자유도 |

## 3. 무엇이 같고 무엇이 다른가

같은 것은 구조다. 둘 다 오차 기반 되먹임이고, LADRC를 전개하면 PID 계수로 환산된다. 다른 것은 실질이다.

| | PID | ADRC |
| --- | --- | --- |
| 외란 대응 | 오차 누적 후 반응(사후) | 외란 추정 후 상쇄(사전) |
| D의 노이즈 | 측정 직접 미분, 증폭 | ESO가 필터·추정, 완화 |
| 파라미터 의미 | 서로 얽힘 | 물리적으로 분리 |
| 튜닝 | 시행착오 | 대역폭 지정, 자동 |

수학적으로 등가 형태로 쓸 수 있다고 해서 같은 것은 아니다. 등가 PID의 계수들이 서로 정교하게 묶여 있고, 그 묶임이 대역폭 파라미터화에서 자동으로 나온다는 것이 차이다. 손으로 PID를 그렇게 튜닝하기는 어렵다. ADRC는 좋은 PID를 체계적으로 뽑는 생성기에 가깝다.

## 4. 시간 영역 설계, 주파수 영역 검증

```mermaid
flowchart LR
    T["시간 영역 설계<br/>ESO·상태 추정"] --> F["주파수 영역 검증<br/>Bode·위상여유"]
```

ADRC는 시간 영역에서 설계되지만, 안정성·강인성·노이즈는 주파수 영역에서 봐야 보인다. 등가 PID나 전달함수로 옮기면 Bode·위상여유를 그대로 쓸 수 있다. 14편에서 이 도구를 쓴다.

## 5. 그럼 그냥 PID 쓰면 되지 않나

잘 튜닝된 PID로 충분한 플랜트라면 ADRC가 필요 없다. 등가이기 때문이다. ADRC가 이기는 곳은 외란과 모델 변화가 크고 빈번한 곳이다. 등가 PID의 계수를 그때그때 최적으로 유지하려면 사람이 계속 다시 튜닝해야 하는데, ADRC는 ESO가 그 역할을 실시간으로 한다. 정적으로는 등가, 동적 적응에서 ADRC 우위다.

## ⚠️ 주의

- 등가성은 선형 ADRC에 대한 것이다. 비선형 원형은 별도다.
- 등가 PID 계수가 손 튜닝으로 재현하기 어렵다는 점이 ADRC의 실질 가치다.

## 📌 정리

- 선형 ADRC는 **설정값 가중과 측정 필터를 갖춘 2자유도 PID**와 등가다.
- 같은 것은 구조, 다른 것은 사후 누적 대 사전 추정, 파라미터의 분리성, 자동 튜닝이다.
- 시간 영역 설계를 주파수 영역에서 검증한다.
- 잘 튜닝된 PID로 되는 곳엔 불필요하고, 외란·변화가 크고 잦은 곳에서 우위다.
